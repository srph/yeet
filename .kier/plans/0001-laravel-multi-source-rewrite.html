Objective

Three things at once, in one rewrite:

  1. Drop trigger.dev. It's a paid vendor doing the work of one queued job. Laravel's queue does this for free on hardware you already rent.
  2. Support more than YouTube. X and Facebook now, others later — which means the schema and the extraction path both stop assuming "youtube".
  3. Move to a stack you actually operate. Forge + Postgres in production, DBngin locally. No Vercel, no separate trigger.dev deploy, no env-var syncing between two dashboards.

Expected outcome: one Laravel app, one deploy target, one .env. Paste a YouTube, X, or Facebook URL; the page looks and behaves exactly as it does today.

Proposed approach

The schema, made source-agnostic

Your rename, applied — plus a source discriminator. Status and format stay as plain strings: constraining them in the DB would be rigid this early, when the set of sources, formats, and states is all still moving. The one structural addition is an index on (source, source_id, format, status), so the dedupe fix has something to lean on.

Schema change 1 of 2 beyond your rename

downloadUrl becomes storage_key. Storing a presigned URL in a column is what created the 1-hour-vs-7-day bug: the URL starts decaying the instant it's written. Store the durable thing (the S3 key) and presign at read time in the API resource. The link is then always fresh for a full hour from when the user actually loads the page, and expires_at becomes the real, honest expiry. The frontend still receives a downloadUrl field and never knows the difference.

Schema change 2 of 2 — expired_at is gone

Expiry becomes a status, not a timestamp. expired_at was a nullable timestamp whose only possible meanings were "no" (NULL) and "yes, at some moment nobody will ever query" — which is a boolean wearing a costume, and duplicated state that status already models. It's dropped in favour of a fifth status value, 'expired', written by the prune command.

That leaves expires_at with exactly one reader in the whole app: the scheduled prune command asking "what's due?" Nothing else branches on it. This is also what kills the original bug at the root — the old code confused expiredAt with expiresAt because the two columns were nearly indistinguishable. Now there's only one.

The tradeoff, stated plainly

The prune runs hourly, so a row can sit at status = 'complete' with expires_at just past — up to an hour before prune flips it to 'expired'. During that window a dedupe hit serves a "technically expired" row. This is harmless: the S3 object is still there (prune is what deletes it), so the link works fine. The alternative — checking expires_at in the dedupe query too — would buy nothing except a second reader of the column you just said should have one. Worth knowing, not worth defending against.

How "adapters" actually work here

Worth being explicit, because it's less code than you'd expect: yt-dlp does the extraction for every source. So an adapter isn't a downloader — it's just URL recognition and metadata normalization. Adding TikTok later is one class and one line in a registry.

NEW app/Sources/Source.phpthe whole adapter contract
interface Source
{
    // Does this adapter own the pasted URL? First match wins.
    public function matches(string $url): bool;

    // The stable per-source identifier, used for the storage key
    // and the (source, source_id, format) dedupe index.
    public function extractId(string $url): ?string;

    // The value stored in the `source` column: 'youtube' | 'x' | 'facebook'.
    public function key(): string;
}
NEW app/Sources/YouTubeSource.phpnote: /shorts/ now included
class YouTubeSource implements Source
{
    // Ported from the three regexes in app/api/download/route.ts:92,
    // PLUS the /shorts/ pattern the old app silently rejected.
    private const PATTERNS = [
        '~youtu\.be/([a-zA-Z0-9_-]{11})~',
        '~[?&]v=([a-zA-Z0-9_-]{11})~',
        '~embed/([a-zA-Z0-9_-]{11})~',
        '~shorts/([a-zA-Z0-9_-]{11})~',   // <- new
    ];

    public function key(): string { return 'youtube'; }

    public function matches(string $url): bool
    {
        return $this->extractId($url) !== null;
    }

    public function extractId(string $url): ?string
    {
        foreach (self::PATTERNS as $pattern) {
            if (preg_match($pattern, $url, $m)) return $m[1];
        }
        return null;
    }
}

NEW app/Sources/SourceResolver.phpregistry — adding a source = one line
class SourceResolver
{
    // Order matters only if two adapters could claim the same URL.
    // They can't today, but keep the most specific first as a habit.
    public function __construct(
        private array $sources = new YouTubeSource, // see providers for real binding
    ) {}

    // Returns [Source, id] or null if nothing recognizes the URL.
    // The controller turns null into a 422 with a field error,
    // which is what the old zod .refine() did.
    public function resolve(string $url): ?array
    {
        foreach ($this->sources as $source) {
            if ($id = $source->extractId($url)) {
                return [$source, $id];
            }
        }
        return null;
    }
}

    // The rename: youtube* -> source*, plus the discriminator.
    $table->string('source');
    $table->text('source_url');
    $table->string('source_id');
    $table->text('source_title');
    $table->text('source_thumbnail')->nullable();  // X posts often lack one

    $table->string('format');
    $table->string('status')->default('queued');

    // The storage-layer pair. `storage_key` is the durable S3 key —
    // NOT a presigned URL, which is what fixes the 1hr-link/7day-expiry
    // mismatch. `storage_file_name` is the name we hand the browser.
    $table->string('storage_key')->nullable();
    $table->string('storage_file_name')->nullable();
    $table->text('reason')->nullable();

    // The ONLY reader of this column is downloads:prune, asking
    // "what's due for deletion?". Nothing else branches on it.
    // There is no expired_at — expiry is a status, not a timestamp.
    $table->timestamp('expires_at')->nullable();
    $table->timestamps();

    $table->index(['created_at']);

    // Backs the dedupe lookup that was broken before.
    $table->index(['source', 'source_id', 'format', 'status']);

    // Backs the prune command's "what's due?" query specifically.
    $table->index(['status', 'expires_at']);
});

The job — trigger.dev's task, translated

The old task put all its DB writes in onStart/onSuccess/onFailure hooks and none in run. Laravel has no onStart/onSuccess hooks, so those writes move inline; onFailure maps cleanly onto the failed() method.

NEW app/Jobs/ProcessDownload.phpreplaces trigger/download-youtube.ts
class ProcessDownload implements ShouldQueue
{
    use Queueable;

    // Mirrors trigger.config.ts: maxAttempts 3, maxDuration 3600.
    public int $tries = 3;
    public int $timeout = 3600;

    // trigger.dev used exponential backoff 1s -> 10s, factor 2, randomized.
    // Laravel takes a plain array of seconds; close enough, and honestly
    // these retries only help against transient network blips.
    public function backoff(): array { return [1, 5, 10]; }

    public function __construct(public Download $download) {}

    public function handle(YtDlp $ytdlp): void
    {
        // Was trigger.dev's onStart hook. Fires per-attempt, same as before.
        $this->download->update(['status' => 'processing']);

        $fileName = "{$this->download->source_id}.{$this->download->format}";
        $key = sprintf('%s/%s/%s',
            config('services.storage.base_directory'),
            $this->download->source,   // partition by source now, not one flat dir
            $fileName,
        );

        // yt-dlp writes straight to a temp file on disk. This is what
        // kills the old Buffer.concat() OOM ceiling - the file never
        // has to fit in memory.
        $tmp = $ytdlp->download(
            url:    $this->download->source_url,
            format: $this->download->format,
        );

        try {
            // putFileAs streams; no full read into RAM.
            Storage::disk('s3')->putFileAs(dirname($key), new File($tmp), basename($key), [
                'ContentType' => $this->download->format === 'mp3'
                    ? 'audio/mpeg'
                    : 'video/mp4',
            ]);
        } finally {
            @unlink($tmp);  // never leave temp files on the Forge box
        }

        // Was onSuccess. Note we store the KEY, not a presigned URL.
        // expires_at is now truthful: the object really does live 7 days,
        // and each presigned link is minted fresh on read.
        $this->download->update([
            'status'            => 'complete',
            'storage_key'       => $key,
            'storage_file_name' => $fileName,
            'expires_at'        => now()->addDays(7),
        ]);
    }

    // Was onFailure. Like trigger.dev, this fires only after all
    // $tries are exhausted - so a row sits in 'processing' during retries.
    public function failed(?Throwable $e): void
    {
        $this->download->update([
            'status' => 'failed',
            'reason' => $e?->getMessage(),
        ]);
    }
}

The yt-dlp wrapper

NEW app/Sources/YtDlp.phpreplaces app/yt.ts + youtubei.js entirely
class YtDlp
{
    // Metadata probe. Replaces `yt.getInfo(videoId)` in the old POST route.
    // --dump-json emits one JSON blob describing the media without
    // downloading it. Same call works for youtube, x, and facebook -
    // that uniformity is the whole reason we picked yt-dlp.
    public function probe(string $url): array
    {
        $result = Process::timeout(30)->run([
            config('services.ytdlp.binary'),
            '--dump-json',
            '--no-warnings',
            '--no-playlist',     // a playlist URL must not fan out
            $url,
        ]);

        throw_unless($result->successful(),
            new SourceUnavailable($result->errorOutput()));

        $json = json_decode($result->output(), true);

        // Normalized across every source. `thumbnail` may be absent
        // (common on X) - hence the nullable column.
        return [
            'title'     => $json['title'] ?? 'Untitled',
            'thumbnail' => $json['thumbnail'] ?? null,
            'duration'  => $json['duration'] ?? null,
        ];
    }

    // Returns a path to a temp file on local disk. Caller uploads + unlinks.
    public function download(string $url, string $format): string
    {
        $dir = storage_path('app/tmp/'.Str::ulid());
        mkdir($dir, 0755, true);
        $out = "{$dir}/media.%(ext)s";   // yt-dlp fills in the real ext

        $args = $format === 'mp3'
            // Audio-only: extract and transcode to mp3. Needs ffmpeg (present).
            ? ['-x', '--audio-format', 'mp3', '--audio-quality', '0']
            // Best video+audio, remuxed to mp4. Also needs ffmpeg to merge
            // the separate streams YouTube serves.
            : ['-f', 'bv*+ba/b', '--merge-output-format', 'mp4'];

        // timeout must stay under the job's $timeout of 3600.
        $result = Process::timeout(3500)->run([
            config('services.ytdlp.binary'),
            '--no-playlist', '--no-warnings',
            ...$args,
            '-o', $out,
            $url,
        ]);

        throw_unless($result->successful(),
            new DownloadFailed($result->errorOutput()));

        // Exactly one file lands in the scratch dir.
        $files = glob("{$dir}/media.*");
        throw_if(empty($files), new DownloadFailed('yt-dlp produced no file'));

        return $files[0];
    }
}

The controller

NEW app/Http/Controllers/DownloadController.phpreplaces both Next.js API routes
class DownloadController
{
    // POST /api/download — same URL and JSON shape the frontend
    // already posts to, so mutations.ts needs no change.
    public function store(Request $request, SourceResolver $resolver, YtDlp $ytdlp)
    {
        // Was zod. Laravel returns 422 with the same field-error shape.
        $data = $request->validate([
            'url'    => ['required', 'url'],
            'format' => ['required', Rule::in(['mp3', 'mp4'])],
        ]);

        $resolved = $resolver->resolve($data['url']);

        // The old app threw an invariant here and 500'd. This is a real 422.
        if (! $resolved) {
            throw ValidationException::withMessages([
                'url' => 'That link is not from a supported site.',
            ]);
        }

        [$source, $sourceId] = $resolved;

        // THE DEDUPE FIX. The original queried `expiredAt` (never written,
        // always NULL) so this branch was dead and every submit re-downloaded.
        //
        // Keying off status alone is what dropping expired_at buys us: a row
        // is reusable unless it's terminal-and-useless. 'queued'/'processing'
        // means someone else already started this exact job - join it rather
        // than dispatch a duplicate. 'complete' means the file is sitting in
        // the bucket. No date math, no NULL handling, no second column to
        // confuse with the first.
        $existing = Download::query()
            ->where('source', $source->key())
            ->where('source_id', $sourceId)
            ->where('format', $data['format'])
            ->whereNotIn('status', ['failed', 'expired'])
            ->first();

        // Returning the model directly — Eloquent serializes it to JSON,
        // snake_case keys and all. No resource layer.
        if ($existing) {
            return $existing;
        }

        $meta = $ytdlp->probe($data['url']);

        $download = Download::create([
            'source'           => $source->key(),
            'source_url'       => $data['url'],
            'source_id'        => $sourceId,
            'source_title'     => $meta['title'],
            'source_thumbnail' => $meta['thumbnail'],
            'format'           => $data['format'],
            'status'           => 'queued',
        ]);

        // Was downloadYoutubeTask.trigger(...). Fire and forget, same as before.
        ProcessDownload::dispatch($download)->onQueue('downloads');

        return $download;
    }

    // GET /api/download/{download} — the 1s polling target.
    // Route-model binding gives a real 404 instead of the old 500.
    public function show(Download $download)
    {
        return $download;
    }
}

The model — no resource layer

Dropping DownloadResource

The controller returns the model and Eloquent serializes it, snake_case and all. The frontend adapts. A resource class that only renamed keys wasn't paying for itself — it's a whole file, plus a second place to edit every time a column changes.

The one thing that did earn its keep there — minting the presigned URL on read — moves onto the model as an accessor, which is where it arguably belonged anyway. Same fix, no extra layer.

NEW app/Models/Download.phpserializes itself; no resource needed
class Download extends Model
{
    use HasUlids, HasFactory;

    protected $guarded = [];

    protected function casts(): array
    {
        return ['expires_at' => 'datetime'];
    }

    // storage_key is an internal detail — the bucket layout is nobody's
    // business but ours, and it's useless to the client anyway.
    protected $hidden = ['storage_key'];

    // download_url isn't a column, so it must be appended explicitly
    // to show up in toJson().
    protected $appends = ['download_url'];

    // The presign-on-read fix, now living on the model. Minted fresh on
    // every serialization, so it's valid 1hr from NOW rather than 1hr
    // from whenever the job happened to finish. This is what stops links
    // dying 6 days before expires_at claims they will.
    //
    // Null after prune nulls storage_key — the object is gone, so there's
    // nothing honest to point at.
    protected function downloadUrl(): Attribute
    {
        return Attribute::get(fn () => $this->storage_key
            ? Storage::disk('s3')->temporaryUrl($this->storage_key, now()->addHour())
            : null);
    }
}

NEW app/Console/Commands/PruneDownloads.phpthe sole consumer of expires_at
class PruneDownloads extends Command
{
    protected $signature = 'downloads:prune';

    public function handle(): void
    {
        // The one place expires_at is ever read. Only 'complete' rows own
        // an S3 object, so nothing else is prunable. chunkById keeps memory
        // flat if a backlog ever builds up.
        Download::query()
            ->where('status', 'complete')
            ->where('expires_at', '<', now())
            ->chunkById(100, function ($downloads) {
                foreach ($downloads as $download) {
                    // delete() is idempotent — a missing object is not an error,
                    // so a half-finished previous run recovers cleanly.
                    Storage::disk('s3')->delete($download->storage_key);

                    // The row is kept as a tombstone rather than deleted: it's the
                    // record that this URL was once fetched, and the dedupe query
                    // skips it via whereNotIn(['failed','expired']).
                    $download->update([
                        'status'            => 'expired',   // was: stamping expired_at
                        'storage_key'       => null,        // the object is gone; don't
                        'storage_file_name' => null,        // pretend otherwise
                    ]);
                }
            });

        // Sweep orphaned scratch files from jobs that were kill -9'd
        // between download and unlink (see the disk risk in §8).
        foreach (File::directories(storage_path('app/tmp')) as $dir) {
            if (File::lastModified($dir) < now()->subDay()->timestamp) {
                File::deleteDirectory($dir);
            }
        }
    }
}
  • Schedule hourly in routes/console.php: Schedule::command('downloads:prune')->hourly().
  • Note storage_key and storage_file_name are nulled on prune, so the model's download_url accessor returns null for free — no presigned URL is minted for an object that no longer exists.
  • This is what stops the bucket growing forever, which is the actual point.
Verify: factory a complete row with expires_at in the past + a real object; run the command; assert the object is gone, status is expired, and storage_key is null. Then assert a fresh POST for that same URL creates a new row rather than returning the tombstone.

Verification & acceptance

Automated

  • Unit — URL matchers per source, including Shorts, legacy twitter.com, and fb.watch.
  • Feature — POST validates and dispatches; unsupported URL → 422; duplicate submit returns the same row without a second dispatch; unknown id → 404; throttle returns 429.
  • Job — with Storage::fake('s3') and a faked YtDlp: happy path lands complete + storage_key; failed() writes reason.
  • Command — prune deletes expired objects, flips status to expired, and nulls storage_key; a subsequent POST for the same URL re-downloads instead of returning the tombstone.

Manual — the things tests can't reach

The automated suite fakes yt-dlp and S3, which means it deliberately cannot catch the two riskiest things: whether yt-dlp actually pulls from each site, and whether the animations survived the port. Both need eyes.

  • Real download × 3 sources × 2 formats = 6 runs, each producing a file that actually plays.
  • The animation set: form → result crossfade, the four status copy states, the button's grid-morph, DecryptedText's scramble, the submit button's sliding double-arrow, the error toast.
  • A deliberately broken URL → failed state → retry link works → polling actually stops (Network tab; this is the fix in §5).
  • An X post with no thumbnail → the placeholder renders instead of a broken image.

Acceptance criteria

  • YouTube, X, and Facebook URLs all download in both formats.
  • The UI is visually indistinguishable from today's, minus the intentional copy change.
  • No trigger.dev, Prisma, Next.js, or youtubei.js anywhere in the tree — and no PLAYER_ID.
  • Re-submitting a fresh URL returns instantly from cache and dispatches nothing.
  • Download links work for the full expires_at window, not 60 minutes.
  • One .env, one deploy, one dashboard.

Risks

yt-dlp gets blocked from a datacenter IP
This is the big one, and it's a genuine regression risk versus today. YouTube increasingly serves "Sign in to confirm you're not a bot" to cloud IPs — and your Forge box is exactly that. It may work fine on your Mac and fail on the server. Mitigate: test from the Forge box early — during Phase 0, not Phase 8. If blocked: yt-dlp's --cookies-from-browser / a cookies file, or a residential proxy. Facebook is the most likely to demand cookies; public X videos are usually fine.
Format parity isn't guaranteed across sources
The mp3/mp4 toggle assumes both always make sense. A silent X video technically has no audio worth extracting. Mitigate: the bv*+ba/b fallback chain handles missing streams gracefully, and a failure surfaces in the existing failed UI. Not worth pre-solving — see Q1.
Disk fills with temp files
A Forge box has far less disk than a Vercel/trigger.dev function pretended to. Videos now land on real disk before upload. A crash between download and unlink leaves the file behind. Mitigate: the finally block covers exceptions, and the prune command sweeps storage/app/tmp for the kill -9 case. Monitor disk on Forge.
Long downloads block a small worker pool
3 workers × one 40-minute video each = a stalled queue, and no Horizon to see it. Trigger.dev auto-scaled; a Forge daemon does not. Mitigate: start at 2–3 workers and watch. If it bites, either add workers or reject over-long media using the duration that probe() already returns (see Q3).
The grid-morph button breaks under Tailwind v4
Arbitrary grid-cols-[…] values combined with group-data-[status=…] variants are the most exotic CSS in the app. Mitigate: Phase 6 checks it explicitly in all four states. Worst case, hand-write the grid transition in @layer — it's ~15 lines of CSS.

Open questions

Q1 · Should the mp3/mp4 toggle stay visible for every source?
Facebook and X videos are usually watched, not converted to audio. Recommendation: leave it alone. The toggle is source-agnostic and a failed extraction already has a UI state. Don't add conditional logic for a case that may never come up.
Q2 · Should the red play badge change per source?
The bg-red-700 badge over the thumbnail is a YouTube visual tell that'll look odd on an X post. Recommendation: yes, but in a follow-up. It's a one-line map from source to a colour, and it's cosmetic — don't let it hold up the migration.
Q3 · Cap media duration?
probe() already returns duration for free, so rejecting a 3-hour upload at POST time is nearly zero work. Today nothing stops someone queueing a 10-hour stream. Recommendation: skip for now — throttle covers the realistic abuse. Revisit if a worker actually stalls.
Q4 · Keep the /api/* URLs, or go full Inertia?
You said Inertia is "just to render the view," so this plan keeps React Query hitting /api/* and the frontend untouched. Assumed: keep as-is. The alternative — router.reload({ only: ['download'] }) — would delete queries.ts/mutations.ts but rewrite the component's data flow, which contradicts "keep almost as is."

Next action

Do Phase 0 before committing to any of this — and specifically, do it on the Forge box, not just the Mac.

The entire plan rests on one bet: that yt-dlp can actually pull from YouTube, X, and Facebook from your production IP. Everything else here is routine Laravel work with a known shape. That one assumption is neither routine nor verified, and it's cheap to test — brew install yt-dlp, then three --dump-json calls locally, then the same three over SSH on Forge.

If all six succeed, the rest of the plan is mechanical and Phase 1 can start immediately. If the Forge box gets bot-checked, that's not fatal — but it changes the plan materially (cookies, or a proxy, or reconsidering the Node sidecar for YouTube specifically), and you want to know that before deleting the Next.js app.
